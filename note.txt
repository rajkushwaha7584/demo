Sir, first I provisioned the AWS infrastructure using Terraform. It created VPC, public/private subnets, route tables, internet gateway, security groups, EC2 instance, IAM account settings, and ECR repository.

After that, because the EC2 public IP was not assigned automatically, I allocated and attached an Elastic IP to the instance. Then I connected to the Ubuntu EC2 using SSH.

On the EC2 server, I installed Docker and deployed the Node MySQL application using the Docker Hub image philippaul/node-mysql-app:02. The app container is running on port 3000 and exposed to the internet through EC2 port 80.

I also deployed a MySQL 8 container for the database. The Node app connects to the MySQL container internally through Docker network. The application is now accessible at http://13.205.157.238.

The database data is stored in a Docker volume. Inside the MySQL container the path is /var/lib/mysql, and on the EC2 host the actual volume path is /var/lib/docker/volumes/.../_data. So container restart will not remove the data unless the Docker volume is deleted.
